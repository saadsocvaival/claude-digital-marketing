# Standard Content Brief Template — CostSage

> Use this template for every long-form piece (pillar, how-to, comparison, customer, whitepaper).
> Owner: Content Lead • v1.0 • 2026-04-28
> Voice contract: every brief inherits `operations/brand/voice-guidelines.md`.

---

## TL;DR (≤60 words)
One paragraph. What this piece is. Who it's for. Why it exists in the calendar this week. The one question it answers.

## ICP / Reader
- **Role:** (e.g., VP Eng at a Series B SaaS, $50–500K/mo AWS bill, 20–80 engineers)
- **Job-to-be-done:** What they hired this article to do.
- **Sophistication level:** Operator (not beginner). Assume FinOps base vocabulary.
- **Mood at the moment of reading:** (frustrated / researching / comparing / ready-to-buy)

## Search intent + keyword cluster
- **Primary keyword:** (1)
- **Search intent:** (informational / commercial-investigation / transactional)
- **Keyword cluster (3–8):** semantically related queries this piece should rank for.
- **Primary entity (Schema.org):** Article / TechArticle / FAQPage. Identify the entity.
- **SERP scan summary:** What ranks today? What angle is missing? What's our differentiated take?

## Outline (H2/H3, with proof or data note per section)
```
H1: <Title — keyword in first half, voice-guideline-compliant>
  Intro (≤120 words, lead with a number)
  H2: <Section 1 — claim>
    H3: subpoints
    Proof: data point / source / [TBD-OPERATOR] flag
  H2: <Section 2 — mechanism>
  H2: <Section 3 — example or playbook>
  H2: <FAQ block — see below>
  H2: <Conclusion + CTA>
```

## FAQ block (mandatory, 3–6 Q&A)
- Drives FAQ schema, captures long-tail intent.
- Each Q is a real query (use Google's "People also ask" + AnswerThePublic).
- Each A: 40–80 words. Direct answer first sentence; nuance after.

## Internal links (3–6, mandatory)
- 1 to a pillar piece (if this is a how-to or comparison)
- 1 to a /product or /pricing page
- 1 to a relevant blog post already on costsage.ai (use sitemap to confirm)
- 1 to whitepaper or newsletter signup

## External links (1–3, authority-building)
- AWS docs / FinOps Foundation framework / reputable analyst piece.
- Avoid linking competitors directly.

## CTA
- **Primary CTA:** Trial signup ("Connect AWS in under 60 seconds") or whitepaper download.
- **Secondary CTA:** Newsletter subscribe.
- **CTA placement:** End of intro (soft) + end of piece (hard).

## Length target
- pillar: 1,500–2,200 words
- how-to: 800–1,200 words
- comparison: 1,200–1,600 words
- customer: 700–1,000 words
- whitepaper: 2,500–3,500 words

## Distribution map (V5 + V6, mandatory)
- V5 channels (≥1): LinkedIn / X / FinOps Foundation Slack / r/aws / HackerNews / AWS partner channels.
- V6 newsletter slot (≥1): Issue lead / Issue secondary / Issue link block.
- Repurposing recipe: see `repurposing-engine.md`.

## Voice checklist (run before submitting)
- [ ] Headline contains a verb of action (find, ship, cut, approve, execute).
- [ ] First 100 words contain a number with unit + timeframe.
- [ ] No banned vocabulary (leverage, unlock, empower, solutions-as-noun, copilot-about-us).
- [ ] No sentence over 28 words.
- [ ] At least one sentence fragment for emphasis.
- [ ] CTA present, twice.
- [ ] All unverified data points marked `[TBD-OPERATOR]`.

## Metadata
- **Slug:** /blog/<keyword-slug>
- **Meta title:** ≤60 chars, primary keyword in first 40
- **Meta description:** 140–155 chars, action verb, value prop, soft CTA
- **OG image:** specify which template
- **Canonical:** self
- **Author:** [TBD-OPERATOR]
