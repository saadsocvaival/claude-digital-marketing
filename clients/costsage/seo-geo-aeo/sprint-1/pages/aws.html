How does CostSage optimise AWS costs?

CostSage connects to AWS through a read-only IAM role in under 60 seconds, then continuously analyses EC2, RDS, EBS, S3, and Savings Plan / Reserved Instance coverage across every linked account. AWS Waste We Find

🖥️ Oversized EC2 & RDS

The agent benchmarks utilisation against 14 days of CloudWatch metrics and recommends a target instance class with ≥99.9% headroom guarantee.

⏸️ Idle Resources

EC2, RDS, ElastiCache, NAT Gateways, and load balancers running below thresholds are flagged for stop/terminate with optional weekend/holiday schedules.

💳 Savings Plan / RI Coverage Gaps

Continuous coverage analysis across compute and database commitments. The agent recommends term, payment option, and quantity per the optimal coverage ratio.

💾 Orphaned EBS Volumes & Snapshots

Detached volumes and stale snapshots older than your retention policy are surfaced for deletion after a one-click human approval.

🏷️ Tag Hygiene & Cost Allocation

Untagged or mistagged resources are flagged against your tagging policy. The agent can apply default tags inside a scoped write role.

AWS-Specific Questions

What AWS services does CostSage cover?

EC2, RDS, EBS, S3, ElastiCache, EKS, Lambda, CloudFront, NAT Gateways, and Savings Plan / Reserved Instance commitments. Tag hygiene and cost-allocation reports are evaluated across every linked account in your AWS Organization.

Can CostSage execute changes on my AWS account?

Only after explicit human approval. Analysis runs against a read-only IAM role. Execution requires you to opt into a scoped write role that limits the agent to specific actions on resources you've pre-approved. Every action is logged.
